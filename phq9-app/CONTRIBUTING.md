# 贡献指南

感谢您有兴趣为心理健康自评量表中心项目做出贡献！

## 🚀 如何贡献

### 1. Fork 项目
1. 点击右上角的 "Fork" 按钮
2. 将项目克隆到您的本地环境
3. 创建一个新的分支用于您的贡献

### 2. 开发环境设置
```bash
git clone https://github.com/your-username/phq9-app.git
cd phq9-app
pnpm install
pnpm dev
```

### 3. 提交更改
1. 确保您的代码通过了所有测试
2. 提交清晰、描述性的提交信息
3. 推送到您的 fork
4. 创建一个 Pull Request

## 📋 贡献类型

### 🐛 错误修复
- 修复应用程序中的错误
- 改善用户体验
- 优化性能

### ✨ 新功能
- 添加新的量表支持
- 改进用户界面
- 增强数据可视化

### 📚 文档
- 改进 README 文件
- 添加代码注释
- 创建使用教程

### 🎨 设计
- 改进用户界面设计
- 优化响应式布局
- 增强可访问性

## 🔧 开发规范

### 代码风格
- 使用 TypeScript 进行类型安全的开发
- 遵循 React 最佳实践
- 保持组件的单一职责原则
- 添加适当的错误处理

### 提交信息
使用清晰的提交信息格式：
```
type: description

feat: 添加新功能
fix: 修复错误
docs: 更新文档
style: 代码格式调整
refactor: 代码重构
test: 添加测试
chore: 维护任务
```

### 分支命名
使用描述性的分支名称：
```
feature/add-new-scale
fix/login-error
docs/update-readme
```

## 🧪 测试

在提交 PR 之前，请确保：
1. 代码通过了 TypeScript 类型检查
2. 应用程序能够正常构建
3. 所有功能正常工作
4. 没有引入新的错误

```bash
# 运行类型检查
pnpm typecheck

# 构建项目
pnpm build

# 启动开发服务器进行测试
pnpm dev
```

## 📋 Pull Request 流程

1. **创建 PR**: 提交清晰描述的 Pull Request
2. **描述更改**: 详细说明您所做的更改
3. **测试**: 确保所有测试都通过
4. **审查**: 等待项目维护者审查
5. **合并**: 审查通过后合并到主分支

## 🆘 需要帮助？

如果您在贡献过程中遇到问题：
1. 查看现有的 Issues
2. 创建新的 Issue 描述您的问题
3. 在讨论区寻求帮助

## 📄 行为准则

- 保持友善和尊重
- 欢迎不同的观点和建设性反馈
- 专注于对项目最有利的事情
- 尊重其他贡献者

感谢您的贡献！🎉
